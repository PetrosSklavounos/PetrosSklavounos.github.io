MACHINE LEARNING PROJECT: PREDICTIVE MAINTENANCE OF ELEVATOR SYSTEMS

Objective

To predict abnormal vibrations in an elevator system located in Brazil using machine learning models. By accurately predicting these abnormalities, we can anticipate maintenance needs, prevent potential system failures, thereby improving the efficiency and safety of the elevator.

Method

  1. Data Acquisition and Cleaning: Public sensor data from an elevator system was used in this project. The dataset initially contained NaN (Not a Number) values that were replaced with zero to clean the data.
  2. Exploratory Data Analysis: The dataset was split into training and test sets with an 80:20 ratio.
  3. Model Selection and Training: Linear Regression, Random Forest Regressor, and Gradient Boosting Regressor models were trained on the dataset.
  4. Model Evaluation and Optimization: The performance of the models was evaluated using the Mean Absolute Error (MAE). Further optimization was achieved by increasing the number of estimators to a point where runs were kept short, but optimized.

Software

Python programming language along with various libraries, including NumPy, Pandas, Scikit-learn, and Matplotlib were used.

Limitations

  1. Assumptions on NaN Values: The project assumes that the NaN values in the vibration data can be replaced with zero, which may not accurately reflect the real-world conditions.Although, it must be said that most sensors who have blanks (NaN) mean that the given sampling frequency doesn't align with the index chosen, meaning that there are just gaps that the sensors didn't measure.
  2. Model Assumptions: The models assume that the relationships between the features and the target variable remain constant over time. In the real world, these relationships may change, requiring the model to be updated or retrained.
  3. Evaluation Metric: The models were evaluated based on a single metric, the Mean Absolute Error. Other metrics, including the Mean Squared Error or R2 score, might provide different perspectives on model performance.
