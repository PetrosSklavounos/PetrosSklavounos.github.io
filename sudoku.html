Algorithm Design Project: Sudoku Solver

KNOWLEDGE DOCUMENTATION FOR FUTURE SELF



Objective

The objective of this project was to develop an algorithm that can solve Sudoku puzzles. The algorithm should be able to take any valid Sudoku grid as input and return the completed grid as output. The project was also an opportunity for me to improve my Python programming skills and to learn more about algorithms and data structures.

Method

The method used to solve the Sudoku puzzles was a backtracking algorithm. The algorithm works by placing numbers in the empty spots in the grid. When it finds a number that fits, it moves on to the next empty spot. If it reaches a spot where no number fits, it goes back to the previous spot and changes the number there. This process continues until the puzzle is solved.

Software

The project made use of several software libraries and tools for various tasks:

  • OpenCV: This computer vision library was utilized for image preprocessing tasks such as line extraction and noise removal, among other image manipulation tasks.
  • Tesseract OCR: An open-source Optical Character Recognition (OCR) engine was employed for character recognition in the Sudoku grids.
  • NumPy: This numerical computing library was integral in handling grid data structures and array operations.
  • Matplotlib: Used as a plotting library, Matplotlib was crucial for visualizing intermediate and final images.
  • PIL (Python Imaging Library): This library was used for opening, manipulating, and saving images throughout the project.

Limitations

The limitations of this project include the time complexity of the backtracking algorithm. The algorithm has to check every possible number for every empty spot in the grid, so it can be slow for larger or more difficult puzzles. Furthermore, the algorithm does not handle invalid Sudoku grids. If the input grid is not a valid Sudoku puzzle, the algorithm will not be able to solve it.
